Git là gì? các lệnh cơ bản của git

Các lệnh git cơ bản

1) git config
Tác dụng: Để set user name và email của bạn trong main conggiguration file.
Cách xài: để kiểm tra tên và kiểu email trong cấu hình dùng git config --global user.name và git config --global user.email. Để set email hoặc tên mới git config --global user.name = "huannd0101" và git config --global user.email = "huannd0101@gmail.com"

2) git init
Tác dụng: Khởi tạo 1 git reponsitory 1 project mới hoặc đã có.
Các xài: git init trong thư mục gốc của dự án.

3) git clone
Tác dungL copy 1 git reponsitory từ remote sourse.
Các xài: git clone <:clone git url:>

4) git status
Tác dụng: Để cack trạng thái của nhưng file bạn đã thay đổi trong thư mục làm việc. VD: tát cả các thay đổi cuối cùng từ lần commit cuối cùng
Cách xài: git status trong thư mục làm việc

5) git add
Tác dụng: thay đổi đến stage/index trong thư mục làm việc.
Các xài: git add

6) git commit
Tác dụng: commit nghĩa là một action để git lưu lại một snapshot của các sự thay đổi trong thư mục làm việc. Và các tập tin, thư mục được thay đổi đã phải nằm trong Staging Area. mỗi lần commit nó sẽ được lưu lại luchj sử chỉnh sửa cửa code kèm theo tên và địa chỉ email của người commit. Người ra trong git bạn cũng có thể khôi phục lại tập tin trong lịch sử commit của nó để chia cho một branch khacsm vì vậy bạn sẽ dễ dàng khôi phục lại các thay đổi trước đó.
Các dùng: git commit -m:"đây là message, bạn dùng để note những thay đổi để sau này dễ dò lại"

7) git push/git pull
Tác dụng: push hoặc pull các thay đổi đến remote. Nếu bạn đã added và committed các thay đổi và bạn muốn đẩy nó lên hoặc remote của bạn đã update và bạn apply tất cả thay đổi đó trên code của mình.

8) git remote
Tác dụng: để check remote/source bạn có hoặc add thêm remote
Cách dungfL git remote để kiểm tra và liệt kê. Và git remote add <:remote_url:> để thêm

9) git add
Tác dụng: để đưa một tập tin vào Staging Area
Cách dùng: git add tên_file hoặc muoons thêm hết file của thư mục thì git add all
